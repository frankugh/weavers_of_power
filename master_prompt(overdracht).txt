UX/flow (battle)

Overzicht toont kaarten (enemies + players) in round order

Je kunt:

enemy toevoegen uit template

quick custom enemy toevoegen (met anon image)

player card toevoegen (geen stats; wel Next)

enemy order wijzigen (up/down)

enemy verwijderen

Turn flow:

“Draw” start enemy turn + trekt kaarten

guard van getrokken kaarten wordt direct toegepast zodat het meteen zichtbaar is

“Next” eindigt actieve enemy turn (zonder auto-draw) en selecteert volgende in round order

“Enemy turn (no draw)” triggert alleen start/end effecten zonder draw

Overzicht toont:

Current draw (als effect tekst: Attack X, guard Y, modifiers)

Last draw (zelfde, maar vorige draw)

Combat model & modifiers

Attack modifiers (attack-only): stab, pierce, magic_pierce, sunder

Status effects (persistent): paralyzed, slowed, burn, poison

Semantiek:

burn/poison: DOT bij start-of-turn, negeert armor/guard

slow: halve movement, verdwijnt end-of-turn

paralyze: draws -1 next turn, verdwijnt end-of-turn (transient)

status effects moeten ook toepasbaar zijn bij 0 dmg

Afspraak over data

Enemy templates in JSON (hp/armor/magic ranges, movement, draws_base, loot table, specials, deck references)

Decks in JSON (core decks + enemy-specific additions)

Enemy instance heeft runtime properties: hp_current/max, armor_current/max, guard_current, deck_state, statuses, last_drawn, loot state.
